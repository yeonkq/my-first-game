# My First Game - Week 5 Practice
## 게임 컨셉 및 기획 단계
기본 아이디어: 고전적인 벽돌깨기 스타일에 RPG와 로그라이크 요소를 결합하는 방향을 탐색했습니다.

핵심 메커니즘 설정: * 캐릭터가 마우스나 키보드 조준을 통해 구체를 자동으로 연사하는 액션 슈팅 방식.

라운드제 도입: 5초마다 상단의 타일들이 한 칸씩 아래로 하강하며 압박감을 주는 시스템.

타일에 수치를 부여하여 여러 번 맞춰야 파괴되는 HP(체력) 시스템 적용.

## 기술적 문제 해결 (디버깅 및 로직 최적화)
물리 엔진 개선: 공이 벽돌 사이에 끼어 무한히 진동하는 현상을 방지하기 위해 X축과 Y축 충돌을 분리하여 처리하고, 충돌 즉시 좌표를 강제로 보정하는 로직을 구현했습니다.

조작 방식 변경: 초기 마우스 조준 방식에서, 게임의 손맛과 정교함을 높이기 위해 좌우 방향키(←, →)를 이용한 각도 조절 방식으로 코드를 수정했습니다.

UI/UX 추가: 유저가 현재 상황을 파악할 수 있도록 상단 중앙에 **라운드 표시(ROUND X)**와 하단에 타일 하강 타이머를 배치했습니다.

## 개발 환경 및 버전 관리
기술 스택: Python 언어와 Pygame 라이브러리를 사용했습니다.

버전 관리: GitHub Desktop을 활용하여 로컬 폴더와 원격 저장소를 연결하고, 코드 수정 사항을 커밋(Commit) 및 푸시(Push)하는 워크플로우를 익혔습니다.
